Introduction To Local And Internet File Management: Part 1

Concept 1. Web pages (html files) are made up of text and nothing else! 

Although they sometimes look like large impressive documents, web pages actually are nothing more than text files. In addition to the text that you read when you view a web page, there is other, hidden, text that tells your web browser (Internet Explorer, Safari, Firefox or whatever) how to display the page, where to find pictures, video, and other media files that may make up the webpage. The pictures and media files are totally separate from the web page itself and are stored in folders. This concept can feel a little strange at first but for you to become a competent web developer you must understand and come to terms with it. 

Concept 2. CSS (Cascading Style Sheets) control the structure and look of a modern website or web page. CSS was invented several years after people began to build websites but has become the de-facto standard for controlling things such as the color, width, and link styles of your site. One of the great things about CSS is that it makes it possible to change literally thousands of pages at once (if you happen to have that big of a website :-) just by altering a few letters in a CSS file.  

Concept 3. The web server is not your computer.

As a student who is taking this course in the 21st century, you live in an era when most computers are always connected to the Internet. Being constantly connected tends to blur the distinction between a local and a remote computer, and you may not be clear about the distinction between files stored on your computer and files placed somewhere else on a web server.

Almost everything that you produce in this course should ultimately make its way to the Internet where it will be accessible through a website that you will create. It is important to understand that the webserver that shares your website with the world and the computer that you use to develop your website are not the same machine. In this class you will develop web pages and ultimately websites on the computer in front of you. Once you have developed a web page or site that you are ready to share with the world, you will move the content to another computer known as a web server that, in this case, is physically located in Chicago. 

Here is how it works. 

  • First, you create files and folders on the local computer that is inches away from you and link them together into a local "website" that ONLY exists on  YOUR computer.  
  • Once everything works well on your local computer you use a type of program known as an FTP client to move the files and folders to another computer in Chicago that functions as a web server, that will make all of your work available to the wide wide world.  
  • When you move files and folders to the web YOU HAVE TO CREATE THE SAME FILE AND FOLDER STRUCTURE THAT EXISTS ON YOUR LOCAL COMPUTER OR IT WILL NOT WORK!

Introduction to HTML, and CSS

Web development can be extremely complex, but the underlying structure of all web pages is nothing more than raw unformatted text that is tied together with a formatting language known as html (HyperText Markup Language). The expression HyperText, comes from the idea that webpages contain hyperlinks that are used to move from one webpage to another. One way to conceive of html is to think of it as the skeleton of a webpage. 

CSS (Cascading Style Sheets) is another key component of web development. While html provides the skeleton of a page, CSS is used to control the format. The various colors of a webpage, the position of images, the way that text reacts when you click, tap, or roll over it; all of these features are typically determined by css.  

But enough talk about abstractions. Let's build a webpage.  The only tool we are going to use is a basic text editor named unitron which you can easily find by searching for it. (please note: there are any number of text editors such as notepad (windows), TextWrangler (Mac), Sublime Text, etc. that will also do the job).  

  1. Open Unitron, create a new document and type:   <html>     </html>

In HTML, anything enclosed in carrots or "brackets" are what we call "tags". These are the building blocks of HTML.  The first building block of a webpage is the <html> tags that encase all other tags and all of the page’s content. 

But why two tags? In most cases, tags come in pairs: an opening tag and a closing tag. Closing tags are always indicated by a / as in </html>. 

  1. In-between the two html tags (<html> and </html>), type: <head> </head> and
    <body>   </body>

Notice again that both these tags are pairs. The head and body are the two main parts of any html document. The "head" section never actually appears on a webpage, but contains settings and other important things that affect the webpage. The "body" section contains all of the actual content (words, pictures, css) that will go on the page.

At this point, your code should now look like this:

  1. At the very top of your document, just above the <html> tag, add <!doctype html>.  Although not strictly necessary, the <!doctype html> is there to tell your browser that your page is written in the very latest version of html: html5. 
  2. Immediately after the <html> tag, add these tags <title>  </title>.  In between the tags, type your name, “my first webpage”, or something of the sort, as in <title>my first webpage  </title>.  The title tags define the text that appears at the very top of your browser’s window.

Something interesting about HTML is that whitespace in within code is completely ignored. It makes no difference if you place your tags one line apart or 20 lines apart. As long as they're in the order you want, the final page will look the same in the browser. Tabbing and spacing your tags can make your code easier to read and to work with.

Absolute Positioning: Formatting the easy way

Those of you who who are sick of having to alter a set of numbers and tinker with html or CSS every time you need to position a picture or div tag will enjoy absolute positioning. When used in conjunction with Dreamweaver, a div tag that is positioned absolutely can literally be dragged around and it will stay put!  The main caveat with absolute positioning is that the div tag is always relative to ‘something’ and, by default, are relative to the upper left corner of your browser’s window. 

Remember a few steps back when you were asked to use relative positioning for CSS rules such as right_side_content? When you apply absolute positioning to a div tag inside of another div tag that does NOT have either relative or absolute positioning strange things happen. The position of the new div tag with absolute positioning becomes relative to the nearest ancestor with either relative or absolute positioning; usually the upper left hand corner of the browser window. Suddenly the picture that seemed to be perfectly positioned when you inserted dragged it to just the location where you wanted it, stays put while the div tags that contain all of your content leave it behind. This concept is a bit hard to understand at first but the bottom line is ALWAYS make sure that the parent div tag has relative (usually) or absolute positioning before you stick a new div tag inside of it with absolute positioning . Whew!

If none of this makes sense yet, hopefully it will become clearer as you proceed through the next few steps.

  6. Now return to the CSS editor and click on absolute_example. If necessary, resize the bottom of the CSS styles window so that you can see the properties of absolute_example. Note the the width and height properties have changed and two additional properties that you didn’t create, left and top, have also appeared. Left and Top are the absolute positioning properties that determine the placement of absolute_content inside of its “parent” tag (which in this case is rightside_content.
